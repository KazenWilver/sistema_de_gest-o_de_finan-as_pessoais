SECÇÃO 8 — FUNCIONALIDADES DO SISTEMA (DETALHE FUNCIONAL)
================================================================================

8.1 AUTENTICAÇÃO (COMPLETA)

  a) Registo:
     - Formulário: nome, email, password, confirmar password
     - Validação frontend: email válido, password >= 8 caracteres, match
     - Validação backend: email único, sanitização de inputs
     - Ao registar: cria user + categorias padrão + conta "Carteira"
     - Retorna JWT + dados do utilizador

  b) Login:
     - Formulário: email, password
     - Verifica password com password_verify() (bcrypt)
     - Retorna JWT + dados do utilizador
     - Guarda token no localStorage

  c) Logout:
     - Remove token do localStorage
     - Redireciona para /auth/login

  d) Recuperação de Senha:
     - Formulário "Esqueci a senha": insere email
     - Backend gera token aleatório, guarda em password_resets com expiração
     - Em contexto académico: simula envio de email (retorna token na resposta
       ou mostra no console do servidor)
     - Formulário "Nova senha": insere token + nova password
     - Backend valida token, actualiza password_hash, marca token como usado

8.2 DASHBOARD (VISÃO GERAL FINANCEIRA)

  Componentes:
  - Summary Widget: 4 cartões com Total Receitas, Total Despesas, Saldo,
    Taxa de Poupança (% do mês)
  - Recent Transactions: lista das últimas 5 transações com categoria,
    valor e data
  - Budget Progress: barras de progresso dos orçamentos activos do mês
    (verde < 60%, amarelo 60-80%, vermelho > 80%)
  - Monthly Chart: gráfico de linha receitas vs despesas (últimos 6 meses)
  - Category Pie: gráfico de pizza com distribuição de despesas por categoria

  Filtro: mês/ano seleccionável

8.3 TRANSAÇÕES (CRUD PRINCIPAL)

  Lista:
  - Tabela com colunas: data, descrição, categoria, conta, tipo, valor
  - Paginação server-side (20 por página)
  - Ordenação por data, valor ou categoria (clicável nas colunas)
  - Filtros: período (de-até), tipo (receita/despesa), categoria, conta
  - Badge colorido para tipo (verde=receita, vermelho=despesa)
  - Badge com ícone e cor da categoria

  Formulário (Criar/Editar):
  - Campos: conta (dropdown), categoria (dropdown filtrado por tipo),
    tipo (receita/despesa), valor, descrição, data, método de pagamento,
    notas (opcional)
  - Validação imediata no formulário (required, min value, etc.)
  - Mesmo componente para criar e editar (detecta :id na rota)

  Eliminar:
  - Dialog de confirmação antes de eliminar
  - Feedback visual de sucesso/erro

8.4 CONTAS (ACCOUNTS)

  - CRUD: criar, listar, editar, eliminar
  - Tipos: Carteira (cash), Banco (bank), Mobile Money, Poupança (savings)
  - Cada conta tem moeda associada
  - Saldo calculado automaticamente (soma transações receita - despesa)
  - Não se pode eliminar conta com transações associadas (RESTRICT ou aviso)

8.5 CATEGORIAS

  - CRUD de categorias personalizadas por utilizador
  - Categorias padrão criadas automaticamente no registo (is_default=1)
  - Ícone e cor personalizáveis para cada categoria
  - Separação por tipo (income / expense)
  - Não se pode eliminar categoria com transações associadas

8.6 ORÇAMENTOS (BUDGETS)

  - Definição de limite por categoria ou global (category_id = NULL)
  - Período: semanal, mensal ou anual
  - Indicador visual de progresso:
    * Verde: < 60% consumido
    * Amarelo: 60-80% consumido
    * Vermelho: > 80% consumido
  - Alerta visual quando > 80% consumido
  - Valor gasto calculado automaticamente a partir das transações

8.7 RELATÓRIOS

  - Gráfico de linha: receitas vs despesas nos últimos N meses
  - Gráfico de pizza: distribuição por categoria (despesas)
  - Tabela de resumo mensal: mês, receitas, despesas, saldo
  - Filtros: período (de-até)

8.8 EXPORTAÇÃO

  - CSV: todas as transações filtradas, com cabeçalhos descritivos
    Formato: Data, Tipo, Categoria, Conta, Descrição, Valor, Moeda
  - PDF: relatório formatado com:
    * Cabeçalho com nome do sistema e período
    * Resumo: total receitas, total despesas, saldo
    * Tabela de transações
    * Gerado com FPDF no backend
  - Ambos descarregados directamente pelo browser (Content-Disposition: attachment)

8.9 CONFIGURAÇÕES

  - Perfil: alterar nome, moeda preferida, idioma
  - Segurança: alterar password (exige password actual)
  - Preferências: tema (claro/escuro), idioma, moeda

8.10 PAINEL DE ADMINISTRAÇÃO (ROLE=ADMIN)

  - Lista de utilizadores: nome, email, data de registo, nº de transações
  - Estatísticas globais: total de utilizadores, total de transações no sistema,
    utilizadores activos no último mês

9.1 FLUXO

      → POST /api/ai/insights

          → AIService.php
              → ReportService::getSummary() [dados financeiros do user]
              → Monta prompt estruturado
              → Chama API Claude (api.anthropic.com/v1/messages)
              → Extrai texto da resposta
      → Retorna insights em JSON

9.2 PROMPT TEMPLATE (AIService.php)

  "Analisa os seguintes dados financeiros pessoais e fornece 3 insights
   accionáveis e concisos em {idioma do utilizador}:

   Período: {período}
   Receitas totais: {total_income} {moeda}
   Despesas totais: {total_expense} {moeda}
   Categorias com mais despesas: {top_categories}
   Orçamentos excedidos: {over_budget_count}

   Responde apenas com os 3 insights, em formato JSON:
   [{"title": "...", "description": "...", "type": "warning|tip|success"}]"

9.3 WIDGET NO DASHBOARD

  - Card com ícone de IA
  - Mostra os 3 insights retornados
  - Botão "Gerar novos insights" para refrescar
  - Estado de loading enquanto a API responde
  - Fallback gracioso se a API falhar (esconde o widget, não quebra o sistema)

SECÇÃO 11 — FLUXO FUNCIONAL DO UTILIZADOR
================================================================================

11.1 FLUXO PRINCIPAL

  1. O utilizador regista-se e faz login
  2. Sistema cria automaticamente categorias padrão + conta "Carteira"
  3. Utilizador pode criar contas adicionais (banco, mobile money, etc.)
  4. Utilizador pode personalizar categorias (criar, editar, eliminar)
  5. Utilizador regista transações (receitas e despesas)
     - Selecciona conta, categoria, tipo, valor, data, descrição
  6. Utilizador cria orçamentos mensais por categoria
  7. Dashboard mostra:
     - Saldo, totais, taxa de poupança
     - Últimas transações
     - Progresso dos orçamentos
     - Gráficos de tendências
  8. Utilizador filtra transações por período, categoria, conta
  9. Utilizador exporta relatórios em CSV ou PDF
  10. Utilizador ajusta idioma (PT/EN) e tema (claro/escuro)

11.2 FLUXO DO ADMINISTRADOR

  1. Admin faz login com credenciais (role=admin atribuído na BD)
  2. Acede ao painel de administração
  3. Visualiza lista de utilizadores do sistema
  4. Visualiza estatísticas globais (utilizadores, transações, etc.)
  5. Também tem acesso ao seu próprio espaço financeiro pessoal
